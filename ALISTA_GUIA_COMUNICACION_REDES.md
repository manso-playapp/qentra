# Guia de comunicacion para redes

> Nombre de trabajo: **Alista**. Esta guia se aplica tambien si el nombre final cambia, porque parte de la estrategia de producto.

## Rol de las redes

Las redes deben instalar una idea: una buena recepcion no se improvisa en la puerta; se prepara antes.

No son solo un canal de inspiracion visual. Deben demostrar criterio operativo, cercania y una forma distinta de pensar la experiencia de invitados.

## A quien se habla

Prioridad: organizadores, productores, planners, salones, agencias, equipos de ceremonial e instituciones que reciben personas de manera recurrente.

Secundariamente: anfitriones finales que buscan una experiencia sin friccion, y equipos operativos que necesitan trabajar con claridad.

## Mensaje rector

> Alista ayuda a organizadores y equipos a preparar cada llegada para que el anfitrion pueda estar presente y cada invitado se sienta esperado.

## Jerarquia para cada pieza

1. Abrir con un problema operativo reconocible.
2. Explicar el mecanismo o criterio que lo resuelve.
3. Mostrar el resultado concreto para el equipo.
4. Cerrar con el beneficio humano.

Ejemplo:

> El problema no empieza en la puerta. Empieza cuando la informacion llega tarde. Centralizar confirmaciones y pendientes permite preparar al equipo antes del evento. El resultado: menos decisiones improvisadas y mas atencion para las personas.

## Pilares de contenido

### 1. Preparar antes

Pendientes, confirmaciones, invitados, acompanantes, necesidades, accesos y excepciones. Mostrar que anticipar es mas valioso que reaccionar.

### 2. Operar con claridad

Roles, criterios compartidos, informacion actualizada y autonomia del equipo. Mostrar el trabajo operativo sin tecnicismos innecesarios.

### 3. Recibir con cuidado

La informacion sirve para contemplar, orientar y recibir mejor, no para vigilar. Mostrar el punto de vista del invitado y del anfitrion.

### 4. Aprender de cada evento

Hablar de indicadores, incidencias, mejoras y lecciones. Usar resultados reales; cuando una pieza sea hipotetica, indicarlo claramente.

### 5. Postura de marca

Defender ideas como: la tecnologia debe desaparecer en la experiencia, el anfitrion no debe ser operador y cada interaccion debe mejorar la siguiente.

## Tono de voz

Claro, sereno, preciso y humano. Hablar como una marca que conoce la operacion y respeta a las personas.

Usar con preferencia: preparar, anticipar, coordinar, contemplar, recibir, claridad, informacion util, equipo, presencia, esperado.

Evitar: control total, vigilancia, procesar personas, seguridad extrema, miedo, caos como recurso vacio, IA como promesa sin funcion comprobable.

## Frases aprobadas para explorar

- Llegá al evento con todo preparado.
- Conocé antes. Prepará mejor.
- Lo que puede resolverse antes no deberia convertirse en una urgencia despues.
- Cada interaccion debe mejorar la siguiente.
- Todo listo para que el anfitrion pueda estar presente.
- Una invitacion crea una expectativa. La preparacion la cumple.
- Preparar cada llegada es cuidar la experiencia.

"Vivi el momento" puede usarse como cierre en eventos sociales o testimoniales, nunca como unica propuesta principal para una audiencia profesional.

## Formatos recomendados

- Carruseles de problema -> preparacion -> resultado.
- Videos breves de flujos de trabajo o estados de preparacion.
- Casos de uso por tipo de evento.
- Checklists utiles para organizadores.
- Historias de incidencias prevenidas, con datos anonimizados o autorizados.
- Testimonios que separen beneficio operativo y beneficio emocional.

## Revision antes de publicar

1. Habla primero de una necesidad real del comprador o equipo?
2. Explica como se logra el resultado?
3. Evita prometer una capacidad inexistente?
4. Protege privacidad y evita lenguaje de vigilancia?
5. Termina con una accion, aprendizaje o beneficio humano concreto?
